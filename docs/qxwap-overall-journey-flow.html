Happy Path หลัก

Guest browse → Login → Product detail → Xwap → Owner accepts → Chat → Shipment proof → Confirm → Credit/recommendation loop

  • ต้องกดได้ทุก CTA
  • state refresh โดยไม่ reload browser
  • owner/non-owner visibility ถูก

Unhappy Path หลัก

Invalid login, upload fail, API fail, offer rejected, session expired, restricted user, no products, empty results

  • มี message และ retry
  • ไม่ล้าง form โดยไม่จำเป็น
  • ไม่ทำปุ่มตาย

Edge Path สำคัญ

User ไม่มีสินค้าแต่ส่ง offer ด้วย message/cash/credit ได้, wanted tag click ไป Shop, follow profile แล้ว feed priority เปลี่ยน

  • message-only offer
  • cash + credit offer
  • profile shop route

Operational Path

CMS จำกัด level 4, migrations fail-fast, cookie/CORS production, GitHub Pages API base, stale cache avoidance

  • account_level field
  • API_BASE normalized
  • service worker/cache safe

Legend + Implementation Rule

Green nodeHappy path ที่ต้องทำให้กดสำเร็จจริงใน product
